## What happened

### 1. RTK — "Rust Token Killer" v0.42.4 installed & wired in
A token-optimizing CLI proxy that transparently rewrites dev shell commands (git, grep, etc.) into compressed output. **Claims 60–90% token savings on dev operations.**

- **Binary:** `/home/mello/.local/bin/rtk` (version `rtk 0.42.4`)
- **Source repo:** `/home/mello/repos/headroom/` (Rust crates + Python SDK; the "headroom" project)
- **Wiring:** registered as a Claude Code **PreToolUse hook** — command `rtk hook claude` in `~/.claude/settings.json`. Transparent rewrite, e.g. `git status` → `rtk git status`, 0 tokens overhead.
- **Docs:** `/home/mello/.claude/RTK.md` (meta-commands: `rtk gain`, `rtk gain --history`, `rtk discover`, `rtk proxy <cmd>`).
- **Safety backup made this session:** `~/.claude/settings.json.pre-rtk.20260719-081556.bak`
- **Watch-out:** name collision — if `rtk gain` fails, a different `rtk` (Rust Type Kit) may be on PATH. Verify with `which rtk`.
- This was a **multi-agent swarm** effort — several Claude sessions coordinating over tmux ("explore-wiring" / "explore-versions" teammates). That coordination churn is what motivated item #2.

### 2. New skill: `cleanup-dead-sessions`
Reaps DEAD/ORPHANED Claude swarm teammate sessions and their leftover tmux servers + stale `claude-swarm-*` sockets under `/tmp/tmux-<uid>/`.

- **Location:** `~/.claude/skills/cleanup-dead-sessions/` (`SKILL.md` + `cleanup_dead_sessions.sh`)
- **Safe by design:** never kills a live session (current or foreign) or a live teammate — only reaps things whose owner/parent PID is already dead. Added an **attached-client guard** so an actively-attached session is never destroyed.
- **Modes:** dry-run by default; `--force` to actually reap.
- **Verified this session:** syntax OK; dry-run correctly protects the live session (socket `/tmp/tmux-1001/claude-swarm-3221766`, `owner = self`) and reports 0 orphans to clean.
