# Cloudflare Tunnel & synai.ai DNS — operating notes

## Tunnel is TOKEN-managed (remote config), NOT a local config.yml
- cloudflared runs with `--token ...`; ingress config lives at the Cloudflare edge, not on disk.
- **Source of truth**: `/home/mello/.cloudflared/tunnel-routes.json` (`ingress[]` = `{service, hostname}`).
- Pushed to CF (edge ingress + DNS CNAME) via `/home/mello/.cloudflared/tunnel-manager.py`.
  - Loads `CLOUDFLARE_API_TOKEN`, `CLOUDFLARE_ACCOUNT_ID`, `CLOUDFLARE_TUNNEL_ID` (tunnel id `9f023bc5-0c1e-4fa7-8d6c-96d22d613966`).
  - `tunnel-routes.pre-graph.json` is a backup snapshot.
- Editing `tunnel-routes.json` alone does nothing until `tunnel-manager.py` pushes it.

## DNS truth: use DoH, not plain dig
- Host resolver is `127.0.0.53` (systemd-resolved) -> router `192.168.2.1`. Plain port-53 dig can be answered locally.
- Ground truth immune to interception: `curl -s "https://1.1.1.1/dns-query?name=<host>&type=A" -H 'accept: application/dns-json'` (Status 0=OK, 3=NXDOMAIN).
- All proxied synai.ai hosts resolve to Cloudflare anycast `172.67.155.56` / `104.21.82.79`.
- **Negative-cache TTL = 1800s (30 min)** — SOA minimum for synai.ai. A brand-new hostname queried BEFORE its record exists gets NXDOMAIN cached for up to 30 min on the client/router. Fix = flush DNS (`resolvectl flush-caches` / `ipconfig /flushdns` / dscacheutil) or wait. This was the whole cause of the vault-graph "site can't be reached" scare — the record was live the whole time.
- Anthropic WebFetch/`getaddrinfo` can hold its own stale negative cache; not authoritative about current state.
- A hostname with a DNS record but NO ingress rule resolves fine but returns Cloudflare **error 1033 / no-route** (not NXDOMAIN).

## Obsidian on the web (2 different things — don't confuse)
- **`obsidian.synai.ai`** = the REAL Obsidian app: LinuxServer/Selkies web desktop container. Ingress `http://obsidian:3000`; host-mapped at `127.0.0.1:8099`. Full menus/editor/plugins, streamed via Selkies (GStreamer WebRTC; page has `universalTouchGamepad.js`/`touch-gamepad-host`). Gated behind **Cloudflare Access** (login at `home-server-mello.cloudflareaccess.com`) — a 302 to that login means the route works, it's just auth. Keep Access ON (editable vault).
- **`vault-graph.synai.ai`** = standalone READ-ONLY graph viz (nodes=notes, edges=[[wikilinks]]). Generated by `/home/mello/.cloudflared/vault-graph-gen.py` -> `/home/mello/.cloudflared/vault-graph/` (log: `vault-graph.log`). Public, no auth. NOT the Obsidian UI.
- `comfyui.synai.ai` -> `http://comfyui:8188` (in routes).
- `mello-garden.synai.ai` has NO record (NXDOMAIN everywhere); the garden service is `garden.synai.ai` -> `http://mello-garden:3000`.
