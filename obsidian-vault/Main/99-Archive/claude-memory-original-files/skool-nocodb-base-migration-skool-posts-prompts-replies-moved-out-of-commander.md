# Skool Tables ‚Üí Dedicated "Skool" Base Migration

The three Skool tables were extracted from the Commander base into a new dedicated "Skool" base. NocoDB has no cross-base move endpoint, so consolidation = recreate + copy + remap + redeploy.

## Key Fact That Made the Remap Safe

`post_id` on Skool Prompts is **written but never queried** ‚Äî `skool-api`'s dedup uses slug-based keys:
- `upsert_post` ‚Üí `slug+group_slug`
- `upsert_prompt` ‚Üí `group_slug+post_slug+prompt_index`
- `get_prompt`/`mark_processed` ‚Üí the prompt's own `Id`
- `get_unprocessed_prompts` ‚Üí `is_processed/generator/group_slug`

So new `Id`s assigned by NocoDB are fine; only the `post_id` foreign key needs remapping (built `posts_id_map = {old_post_Id: new_post_Id}` from insert responses).

## Cutover Order (downtime = Tasks 3 ‚Üí 6 only)

1. Phase 0: read-only baseline (row counts 257/532/0, full schema capture, JSON export to `/tmp/skool-migration/{posts,prompts,replies}.json`)
2. Create "Skool" base ‚Üí `nocodb-agent` `POST /api/v2/meta/bases`
3. Recreate three tables in new base ‚Üí `nocodb-agent` `POST /api/v2/meta/bases/{id}/tables`, data columns only (system cols auto-create)
4. **`docker stop skool-api`** (writer halted; old tables frozen)
5. Copy Posts ‚Üí Prompts (with `post_id` remap) ‚Üí Replies (skip, empty)
6. Edit `database.py`, `docker-compose.yml`, `wall_art_pipeline/config.py`
7. `docker stop && rm && compose up -d --build skool-api` via deploy-agent (`--build` required; `database.py` is in the image build context, `docker restart` won't pick it up)
8. Verify (`docker ps healthy`, `curl /health`, `curl /prompts?limit=5`, `docker logs --tail 50`, scheduler messages, no 4xx/5xx)
9. Rename Commander originals to `... (OLD - migrated)`; delete on/after **2026-05-23**

## Why Surgical Recreate Beat "Duplicate-and-Prune"

Commander holds ~30 unrelated tables (TikTok, Etsy tracker, Restore, Idea Pipeline, etc.). Duplicating the whole base doubles blast radius, leaves dozens of tables to delete, reassigns IDs for every other project's tables. Recreating 3 tables touches nothing else.

## SingleSelect Recreate Detail

For Skool Replies, the two SingleSelects must be recreated with `colOptions.options` matching every option title + color from Phase 0 capture: `trigger` (3 opts), `status` (4 opts). Otherwise filter UI breaks.

## NocoDB Client Limitation

`/home/mello/commander/tools/nocodb_client.py` has only single-record CRUD ‚Äî no base/table creation, no bulk insert. All schema operations went through **`nocodb-agent`** (project rule). Container rebuild went through **`deploy-agent`**.

## Out of Scope

- Adding formal `LinkToAnotherRecord` columns to formalize post‚Üîprompt relationship. App relies on implicit slug/`post_id` matching; schema redesign is a separate request.
